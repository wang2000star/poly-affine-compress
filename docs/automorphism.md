# 布尔函数仿射约化：理论框架与算法设计

## 1. 问题定义

给定向量布尔函数 F: F₂ⁿ → F₂ᵏ，寻找仿射变换

```
z = Mx ⊕ b,    M ∈ F₂^{m×n}, b ∈ F₂^m
```

使得存在 G: F₂^m → F₂ᵏ，满足对任意 x ∈ F₂ⁿ：

```
G(Mx ⊕ b) = F(x)                    (1)
```

目标是极小化 G 的 ANF 单项式总数（或各分量 ANF 的并集规模）。

---

## 2. 预备知识：布尔函数的 ANF

### 2.1 代数标准型

布尔函数 f: F₂ⁿ → F₂ 唯一表示为：

```
f(x) = ⨁_{S ⊆ [n]} a_S · ∏_{j∈S} x_j,    a_S ∈ F₂
```

其中乘积是 AND（F₂ 中的乘法），⨁ 是 XOR（F₂ 中的加法）。系数向量 a 的 1 的个数 = T(f)。

### 2.2 Möbius 变换（小规模 n ≤ 25）

真值表 f: {0,1}^n → {0,1} 到 ANF 系数 a 的变换：

```
a_T = ⨁_{x ⊆ T} f(x),     ∀T ⊆ [n]
```

其中 x ⊆ T 表示 supp(x) ⊆ T。在 bit 层面，这等价于：

```
for i = 0..n-1:
  for x in 0..2ⁿ-1:
    if x & (1<<i):
      f[x] ^= f[x ^ (1<<i)]
```

这就是**快速 Möbius 变换**（Fast Moebius Transform），是 Walsh-Hadamard 变换在布尔半环上的对应。复杂度 O(n·2ⁿ)。

### 2.3 Möbius 的逆变换

```
f(x) = ⨁_{T ⊆ x} a_T

for i = 0..n-1:
  for x in 0..2ⁿ-1:
    if x & (1<<i):
      a[x] ^= a[x ^ (1<<i)]    ← 从 a 恢复 f 的 Möbius 与从 f 到 a 的变换相同
```

**性质：** Möbius 变换是**自逆**的（在 GF(2) 上）：对 truth table 做两次 Möbius 得到原值。

### 2.4 真值表表示（n ≤ 20）

对 n ≤ 20，2ⁿ ≤ 1,048,576，真值表可存入内存（1 MB）。用 64-bit word packing：

```
n_words = ceil(2ⁿ / 64)
tt[oi][w] = 第 oi 个输出、第 w 个 word（64 个连续 x 值）
```

**Möbius 的 word-level 实现：**

```
// step i (i < 6): 在 word 内部处理
if i < 6:
  for w in 0..n_words-1:
    tt[w] ^= (tt[w] & mask_lo) << step  // 需要分高低位处理

// step i (i ≥ 6): 跨 word 处理  
if i ≥ 6:
  step_words = 1 << (i-6)
  for w in 0..n_words-1:
    if w & step_words:
      tt[w] ^= tt[w ^ step_words]
```

**SIMD 加速：** AVX2（256-bit）或 AVX-512（512-bit）可以一次处理 4 或 8 个 64-bit word，吞吐量提升 4-8×。

### 2.5 n=21~25 的大规模

当 21 ≤ n ≤ 25（2ⁿ = 2M ~ 32M words，内存 16 MB ~ 256 MB），仍可做 Möbius，但时间显著增加：

| n | 2ⁿ | 内存 | Möbius 时间（估计） |
|---|------|------|-------------------|
| 20 | 1,048,576 | 1 MB | 0.01 s |
| 22 | 4,194,304 | 4 MB | 0.05 s |
| 24 | 16,777,216 | 16 MB | 0.2 s |
| 25 | 33,554,432 | 32 MB | 0.4 s |

对 k 个输出，总时间 = k × 单输出时间。n=25, k=32 可能需要 10-30 秒。

**结论：** n ≤ 20 可完全自由使用 TT+Möbius；n=21~25 可接受；n ≥ 26 需要替代方法。

## 3. 大规模实例的 ANF 分析（n ≥ 26）

对 n ≥ 26，无法直接计算完整 ANF。但我们可以用**不变量**来缩小搜索空间。

### 3.1 依赖集检测（哪些 x_j 影响 f）

**命题：** x_j 影响 f 当且仅当 e_j ∉ Aut(f)。

**算法（采样法）：**

```
function depends(f, j, N=200):
    for t = 1..N:
        x ← random F₂ⁿ
        if f(x) ≠ f(x ⊕ e_j): return True
    return False
```

**可靠性分析：** 每个测试点有至少 50% 概率发现依赖（若 x_j 确实影响 f）。N=200 给出假阴性率 < 2^{-200} ≈ 0。

**SAT 精确法（替代）：**

```
SAT(∃x: f(x) ≠ f(x ⊕ e_j))
```

编码：两份电路 + 一组 XOR 门形成 x' = x ⊕ e_j。对 n=32, 100+ 门电路，求解时间 < 1ms。

### 3.2 双变量依赖检测（哪些 x_i, x_j 同时出现）

检测两个变量是否在同一个 ANF 项中出现：

```
function pair_depends(f, i, j, N=500):
    for t = 1..N:
        x ← random F₂ⁿ
        // 翻转 i 和 j 两个 bit
        x' = x ⊕ (1<<i) ⊕ (1<<j)
        if f(x) ≠ f(x'): return True
    return False
```

**不足：** 无法区分 "x_i·x_j" 项和 "x_i ⊕ x_j" 项（即一次项和二次项都导致值变化）。需要用更精细的代数方法。

### 3.3 代数次数检测

**定理：** 设 f 的 ANF 系数为 a_S。deg(f) < d 当且仅当对所有 |S| = d 的集合 S，有：

```
⨁_{x ⊆ S} f(x) = 0
```

其中 x ⊆ S 遍历 F₂ⁿ 中所有 supp(x) ⊆ S 的点。

**证明：** 由 Möbius 公式 a_S = ⨁_{x ⊆ S} f(x)。若 deg(f) < d，则对所有 |S| ≥ d 有 a_S = 0。□

**算法（检测 deg(f) ≥ d）：**

```
function degree_ge(f, d, N=100):
    for t = 1..N:
        S ← random subset of [n], |S| = d
        // 枚举所有 x ⊆ S (最多 2^d 个点)
        xor_sum = 0
        for each x in F₂ⁿ with supp(x) ⊆ S:
            xor_sum ^= f(x)
        if xor_sum ≠ 0: return True    // a_S ≠ 0 → deg ≥ d
    return False                        // 可能 deg < d
```

**问题：** 枚举 2^d 个点。对 d=10 是 1024 次求值，d=15 是 32768 次。对 hd01/hd02/hd10（100+ 门电路），d 通常 ≤ 5，所以可行。

**实际优化：** 只用一次 SAT 调用代替枚举（当 d 较大时）：

```
SAT(∃x, S: |S| = d, [⨁_{y ⊆ S} f(y)] ≠ 0)
```

但直接编码需要 2^d 个电路实例，复杂度过高。因此建议对 d > 8 用采样法，d ≤ 8 用 SAT 或枚举。

### 3.4 基于电路的次数上界

对电路表示的 f，次数上界可以直接从电路中读出：

```
// 对每个门:
//  AND: deg(output) = deg(input1) + deg(input2)
//  XOR: deg(output) = max(deg(input1), deg(input2))
//  NOT: deg(output) = deg(input)
//  MUX: deg(output) = max(deg(sel)+deg(a), deg(sel)+deg(b), deg(a), deg(b))
```

**直接传播**得到每个输出分量的精确次数（假设 AND 是 F₂ 中的乘法，XOR 是加法）。这种传播在 O(gates) 时间内完成，完全免费。

### 3.5 完整的 ANF 重建（大规模）

已知依赖集 S 和次数上限 d 后，可以在 |S| 维子空间上重建 ANF：

```
1. 确定依赖集: dep_set = {j | f depends on x_j}
2. 确定次数: deg = degree_estimate(f, circuit_analysis)
3. 取 n' = min(|dep_set|, 一些可处理的值)
   若 n' ≤ 25: 在 dep_set 的变量上穷举重建 ANF
   若 n' > 25: 只保留 T=1 的单变量项（系数为 1 的变量）
              + SAT 检测高次项的存在性
```

## 4. 仿射变换的纤维丛理论

### 4.1 变换的几何结构

仿射变换 φ(x) = Mx ⊕ b 把 F₂ⁿ 划分为 ker(M) 的陪集（纤维）：

```
x ∼ x'  ⇔  φ(x) = φ(x')  ⇔  x ⊕ x' ∈ ker(M)
```

纤维数量 = 2^{rank(M)}，每条纤维大小 = 2^{n - rank(M)}。

### 4.2 G 存在的充要条件

**定理 1（常值纤维条件）：**

```
∃G: G∘φ = F  ⇔  ∀Δ ∈ ker(M), ∀x: F(x ⊕ Δ) = F(x)
```

**等价表述：**

```
∃G  ⇔  ker(M) ⊆ Aut(F),  其中
Aut(F) = {Δ ∈ F₂ⁿ | ∀x: F(x ⊕ Δ) = F(x)}
```

### 4.3 一致性验证算法

给定 M 和 truth table F_tt：

```
function is_consistent(M, F_tt, n):
  // 计算 ker(M) 的一组基
  basis = nullspace_basis(M, n)    // 高斯消元, O(n³)
  
  for each Δ in basis:
    // 检查 F 在 Δ 方向是否恒定
    // 即: 对所有 x, F_tt(x) == F_tt(x ⊕ Δ)
    if not constant_in_direction(F_tt, Δ, n):
      return False
  return True
```

`constant_in_direction` 的 TT 实现（n ≤ 20）：

```
function constant_in_direction(F_tt, Δ, n):
  n_words = ceil(2ⁿ / 64)
  for w = 0..n_words-1:
    if F_tt[w] != xor_shift(F_tt, Δ, w):
      return False      // 存在 x 使 f(x) ≠ f(x⊕Δ)
  return True
```

对大 n，用采样法代替全扫描。

### 4.4 不变子空间 Aut(F)

**定义：** Aut(F) 是 F₂ⁿ 的子空间，包含所有使 F 平移不变的 Δ。

**计算（小规模 n ≤ 20，Walsh 功率谱法）：**

```
对每个输出 f_i:
  1. Ŵ = Walsh-Hadamard(f_i)            O(n·2ⁿ)
  2. S[α] = Ŵ[α]² (为防溢出用浮点)      O(2ⁿ)
  3. r = Inverse_Walsh(S)               O(n·2ⁿ)
  4. Aut(f_i) = {Δ | r[Δ] = 1 - ε}      O(2ⁿ)

Aut(F) = ∩_i Aut(f_i)
```

**计算（大规模 n ≥ 26，SAT 法）：**

```
对每个输出 f_i:
  对一组候选 Δ（e_j 和随机组合）:
    SAT(∃x: f_i(x) ≠ f_i(x ⊕ Δ))
    若 UNSAT → Δ ∈ Aut(f_i)

  // 求得 Aut(f_i) 的一组基:
  // 从 {e_0,...,e_{n-1}} 开始, 检查每个是否在 Aut(f_i) 中
  // 再用基扩充法找完整子空间
```

### 4.5 性质：一致性的单调性

**定理 2：** 设 ker(M₁) ⊆ ker(M₂)。则：
- M₂ 一致 ⇒ M₁ 一致
- M₁ 不一致 ⇒ M₂ 不一致

**推论：**
- 加行（缩小核）保持一致性
- 删行（扩大核）可能破坏一致性
- 从 identity 出发（ker={0}，始终一致），加任何行都一致

### 4.6 维数下界

**定理 3：** m ≥ n - dim Aut(F)。下界可达当且仅当 ker(M) = Aut(F) 且 m = n - dim Aut(F)。

## 5. 搜索 M 使 G 存在的基本策略

### 5.1 完全搜索不可行

对 n=32，可能的 M 数量约为 Σ_{m=1}^{32} C(2³², m) ≈ 2^{32×32} = 2^{1024}。穷举不可行。

### 5.2 降维原则

以下条件确保 ker(M) ⊆ Aut(F)，即 G 存在：

```
(1) ker(M) = {0}          ← M 列满秩, m=n 且 rank=n
(2) M 的行向量 ⊆ V        ← V 是 Aut(F) 补空间的行表示
(3) ker(M) = Aut(F)       ← 最优压缩, 达到下界
```

**实际操作：** 取 Aut(F) 的补空间 V（dim n' = n-d），在 V 上搜索 M 的行。

### 5.3 渐进式构造

```
M_curr = []    // 空
basis = {e_0,...,e_{n-1}}  // ker(M_curr) 的基

while |M_curr| < max_m:
  对每个候选行 w:
    // 步骤 1: 排他性检查
    if rank(M_curr ∪ [w]) == rank(M_curr):
      continue          // w 的行向量在 M_curr 的行空间中，冗余
    
    // 步骤 2: 找 Δ 使 M_curr·Δ=0, w·Δ=1
    增广矩阵 [M_curr; w]·Δ = [0; 1]
    高斯消元求特解 Δ
    if 无解: continue   // w 不提供新方向
    
    // 步骤 3: 检查 Δ 是否在 Aut(F) 中
    if Δ ∈ Aut(F): continue  // w 划分了无效方向
    
    // 步骤 4: 接受 w
    M_curr ∪= [w]
    更新 ker(M_curr) 的基
```

### 5.4 候选行评分

对通过排他性检验的候选行 w，需要评分选择加入：

```
// Walsh 评分（已有）
score = Σ_{oi} |W_oi(w)|  // 各输出在 w 方向的 Walsh 幅值之和

// 差分评分（新）
score = Σ_{oi} Prob_x[f_oi(x) ≠ f_oi(x ⊕ Δ)]
      其中 Δ 是步骤 2 解出的特解（M_curr·Δ=0, w·Δ=1）

// 稀疏性评分（直接估计 T(g)）
score = -E[T(g_oi)]       // 估计新 ANF 的单项式数的变化
```

实际中 Walsh 评分最实用——可直接从 Walsh 谱得到，无需额外电路求值。

## 6. 优化 g 的 ANF 稀疏性

### 6.1 T(g) 的决定因素

对单输出 f 和变换 z = Mx⊕b (m=n)，g(z) = f(M^{-1}(z⊕b))。T(g) = T(f)（Möbius 项数在可逆变换下不变——实际上这不对。T(f) 是 ANF 项数，而 ANF 项与变量命名有关，可逆仿射变换会改变哪些变量名但项数不变）。

等等，仔细想：如果 M 可逆，g(z) = f(M^{-1}(z⊕b))，g 的 ANF 是 f 的 ANF 做变量代换 x = M^{-1}z⊕(M^{-1}b)。变量代换不改变项数（每个单项式映射到另一个单项式，次数不变，一一对应），所以 T(g) = T(f)。

**所以可逆 M 无法压缩 ANF。真正的压缩只来自 m < n（投影退化）。**

### 6.2 互补对滤波

m > n 时，如果 z_i 和 z_j 是互补对（z_i = x_k, z_j = x_k ⊕ 1），则 z_i·z_j = 0（因为 x_k(x_k⊕1) = 0）。在莫比乌斯变换后，所有包含互补对的项都可以滤除。

**实现：**

```
1. 对 g 做 Möbius → ANF 系数 a
2. 对每个互补对 (i,j):
   删除所有包含 i 和 j 的项
3. 对剩余系数做逆 Möbius → 新的 g_tt
```

这等价于在 ANF 系数层面施加约束 z_i·z_j = 0。滤除后 T(g) 可能大幅下降。

### 6.3 依赖集缩减

对策略 2（每输出独立 M），筛选仅保留依赖集内的输入变量：

```
M_i 的行只在 D_i = {j | e_j ∉ Aut(f_i)} 内有 1。
```

此时各行支撑集在 n' = |D_i| 维上。T(g) 的上界显著降低。

### 6.4 联合 vs 独立策略的权衡

**策略 1（联合 M）：** 所有输出共享 M
- Aut(F) = ∩ Aut(f_i)，更小
- m 下界更大
- 各输出互相影响，T(g) 可能增大

**策略 2（独立 M_i）：** 各输出有各自 M_i
- Aut(f_i) 各自，更大
- 各 m_i 下界更小
- 合并后 z 是各 m_i 之和，可能很大

**混合：** 分组共享（共享部分 M 行）

## 8. 已知问题的理论分析

### 8.1 hd08 (n=8, k=1)

- Aut(F): Walsh 法计算，假设 dim=0（非线性函数无平移不变性）
- m_min = 8 - 0 = 8，但已知存在压缩到 m=6
- **矛盾解释：** m < n 时 rank(M) ≤ m。若 rank(M)=6，ker(M) 维数=2。一致性需 ker(M) ⊆ Aut(F)。但 Aut(F) 维数=0 意味着 ker(M) 必须为 {0}，即 rank(M)=8。
  
  **结论：** hd08 的 m=6 变换必须是**不一致的**（G 不完全等于 F 的某个分量），或在某个我们没有考虑的近似意义下成立。

  等等，但这不可能。我们的代码中 `evaluate_Mb_single_output` 会检查一致性，不一致时返回 INT64_MAX。hd08 结果确实是通过验证的。

  再检查：hd08 的 m=6 解是 `hill_climb` 从 identity 出发找到的。但 hill_climb 用的是 `evaluate_Mb` 中的 `hill_climb`，它作用于 m=n 的 M（修改 M 的行但保留 m=n）。所以 hill climb 后的 M 仍是 m=n，只是某些行可能变成全 0 → rank < 8。

  m=n=8，但 rank(M)=6？不对，m 仍为 8。如果 rank(M)=6，则 ker(M) 维数=2。一致性要求这 2 维在 Aut(F) 中。但 Aut(F) 维数=0...

  实际上，对 hd08 函数（8 输入单输出），Aut(F) 可能不是 {0}。让我重新思考：如果 Aut(F)={0}，那么任何非零 Δ 都会改变至少一个 x 处的 f 值。对于完全随机函数确实如此。但 hd08 是一个有结构的电路函数（39 门），可能具有非平凡的不变性。

  或者更可能的解释：hill climb 从 identity (m=n=8) 出发，修改了 M 的行但维持 m=8。得到的变换具有 ker(M)={Δ|MΔ=0}。如果 ker(M) 的维数 > 0，则需 ker(M) ⊆ Aut(F)。也许 hd08 本身就有 Aut(F) ≠ {0}。

  关键点：**我们之前从未计算过实际的 Aut(F) 值**。这只是理论分析。

### 8.2 hd01 (n=32, k=32)

- 已知结果：每个输出 T=1（常数或单变量）
- 对输出 f_i 仅依赖一个 x_j：Aut(f_i) 包含所有与 e_j 正交的向量（dim = 31）
- m_min = 32 - 31 = 1
- Python opt2 排列 M 在可逆变换下恰好保持了每输出单变量的结构
- 联合 Aut(F) = ∩_i Aut(f_i)。如果各输出依赖不同的变量：
  - 每个 Aut(f_i) 排除 e_j（该输出依赖的 j）
  - 对 k=32 个输出，若覆盖所有 32 个变量：∩ Aut(f_i) = {0}
  - 即联合策略下 Aut(F)={0}，m_min=32，无法压缩

### 8.3 hd10 (n=32, k=32)

- 2 个非平凡输出（其余 30 个为常数 f_i(x) = 0）
- 对常数输出：Aut(f_i) = F₂³²（全空间），dim=32
- 对非平凡输出：依赖少量输入变量
- 联合 Aut(F) = Aut(f_nonconst1) ∩ Aut(f_nonconst2)，dim 可能仍接近 32
- 因此联合策略可行，在接近全空间不变的条件下实现压缩
