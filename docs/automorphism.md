# 布尔函数仿射约化：理论框架与算法设计

## 1. 问题定义

给定向量布尔函数 F: F₂ⁿ → F₂ᵏ，寻找仿射变换

```
z = Mx ⊕ b,    M ∈ F₂^{m×n}, b ∈ F₂^m
```

使得存在 G: F₂^m → F₂ᵏ，满足对任意 x ∈ F₂ⁿ：

```
G(Mx ⊕ b) = F(x)                    (1)
```

目标是极小化 G 的 ANF 单项式总数（或各分量 ANF 的并集规模）。

---

## 2. 预备知识：布尔函数的 ANF

### 2.1 代数标准型

布尔函数 f: F₂ⁿ → F₂ 唯一表示为：

```
f(x) = ⨁_{S ⊆ [n]} a_S · ∏_{j∈S} x_j,    a_S ∈ F₂
```

其中乘积是 AND（F₂ 中的乘法），⨁ 是 XOR（F₂ 中的加法）。系数向量 a 的 1 的个数 = T(f)。

### 2.2 Möbius 变换（小规模 n ≤ 25）

真值表 f: {0,1}^n → {0,1} 到 ANF 系数 a 的变换：

```
a_T = ⨁_{x ⊆ T} f(x),     ∀T ⊆ [n]
```

其中 x ⊆ T 表示 supp(x) ⊆ T。在 bit 层面，这等价于：

```
for i = 0..n-1:
  for x in 0..2ⁿ-1:
    if x & (1<<i):
      f[x] ^= f[x ^ (1<<i)]
```

这就是**快速 Möbius 变换**（Fast Moebius Transform），是 Walsh-Hadamard 变换在布尔半环上的对应。复杂度 O(n·2ⁿ)。

### 2.3 Möbius 的逆变换

```
f(x) = ⨁_{T ⊆ x} a_T

for i = 0..n-1:
  for x in 0..2ⁿ-1:
    if x & (1<<i):
      a[x] ^= a[x ^ (1<<i)]    ← 从 a 恢复 f 的 Möbius 与从 f 到 a 的变换相同
```

**性质：** Möbius 变换是**自逆**的（在 GF(2) 上）：对 truth table 做两次 Möbius 得到原值。

### 2.4 真值表表示（n ≤ 20）

对 n ≤ 20，2ⁿ ≤ 1,048,576，真值表可存入内存（1 MB）。用 64-bit word packing：

```
n_words = ceil(2ⁿ / 64)
tt[oi][w] = 第 oi 个输出、第 w 个 word（64 个连续 x 值）
```

**Möbius 的 word-level 实现：**

```
// step i (i < 6): 在 word 内部处理
if i < 6:
  for w in 0..n_words-1:
    tt[w] ^= (tt[w] & mask_lo) << step  // 需要分高低位处理

// step i (i ≥ 6): 跨 word 处理  
if i ≥ 6:
  step_words = 1 << (i-6)
  for w in 0..n_words-1:
    if w & step_words:
      tt[w] ^= tt[w ^ step_words]
```

**SIMD 加速：** AVX2（256-bit）或 AVX-512（512-bit）可以一次处理 4 或 8 个 64-bit word，吞吐量提升 4-8×。

### 2.5 n=21~25 的大规模

当 21 ≤ n ≤ 25（2ⁿ = 2M ~ 32M words，内存 16 MB ~ 256 MB），仍可做 Möbius，但时间显著增加：

| n | 2ⁿ | 内存 | Möbius 时间（估计） |
|---|------|------|-------------------|
| 20 | 1,048,576 | 1 MB | 0.01 s |
| 22 | 4,194,304 | 4 MB | 0.05 s |
| 24 | 16,777,216 | 16 MB | 0.2 s |
| 25 | 33,554,432 | 32 MB | 0.4 s |

对 k 个输出，总时间 = k × 单输出时间。n=25, k=32 可能需要 10-30 秒。

**结论：** n ≤ 20 可完全自由使用 TT+Möbius；n=21~25 可接受；n ≥ 26 需要替代方法。

## 3. 大规模实例的 ANF 分析（n ≥ 26）

对 n ≥ 26，无法直接计算完整 ANF。但我们可以用**不变量**来缩小搜索空间。

### 3.1 依赖集检测（哪些 x_j 影响 f）

**命题：** x_j 影响 f 当且仅当 e_j ∉ Aut(f)。

**算法（采样法）：**

```
function depends(f, j, N=200):
    for t = 1..N:
        x ← random F₂ⁿ
        if f(x) ≠ f(x ⊕ e_j): return True
    return False
```

**可靠性分析：** 每个测试点有至少 50% 概率发现依赖（若 x_j 确实影响 f）。N=200 给出假阴性率 < 2^{-200} ≈ 0。

**SAT 精确法（替代）：**

```
SAT(∃x: f(x) ≠ f(x ⊕ e_j))
```

编码：两份电路 + 一组 XOR 门形成 x' = x ⊕ e_j。对 n=32, 100+ 门电路，求解时间 < 1ms。

### 3.2 双变量依赖检测（哪些 x_i, x_j 同时出现）

检测两个变量是否在同一个 ANF 项中出现：

```
function pair_depends(f, i, j, N=500):
    for t = 1..N:
        x ← random F₂ⁿ
        // 翻转 i 和 j 两个 bit
        x' = x ⊕ (1<<i) ⊕ (1<<j)
        if f(x) ≠ f(x'): return True
    return False
```

**不足：** 无法区分 "x_i·x_j" 项和 "x_i ⊕ x_j" 项（即一次项和二次项都导致值变化）。需要用更精细的代数方法。

### 3.3 代数次数检测

**定理：** 设 f 的 ANF 系数为 a_S。deg(f) < d 当且仅当对所有 |S| = d 的集合 S，有：

```
⨁_{x ⊆ S} f(x) = 0
```

其中 x ⊆ S 遍历 F₂ⁿ 中所有 supp(x) ⊆ S 的点。

**证明：** 由 Möbius 公式 a_S = ⨁_{x ⊆ S} f(x)。若 deg(f) < d，则对所有 |S| ≥ d 有 a_S = 0。□

**算法（检测 deg(f) ≥ d）：**

```
function degree_ge(f, d, N=100):
    for t = 1..N:
        S ← random subset of [n], |S| = d
        // 枚举所有 x ⊆ S (最多 2^d 个点)
        xor_sum = 0
        for each x in F₂ⁿ with supp(x) ⊆ S:
            xor_sum ^= f(x)
        if xor_sum ≠ 0: return True    // a_S ≠ 0 → deg ≥ d
    return False                        // 可能 deg < d
```

**问题：** 枚举 2^d 个点。对 d=10 是 1024 次求值，d=15 是 32768 次。对 hd01/hd02/hd10（100+ 门电路），d 通常 ≤ 5，所以可行。

**实际优化：** 只用一次 SAT 调用代替枚举（当 d 较大时）：

```
SAT(∃x, S: |S| = d, [⨁_{y ⊆ S} f(y)] ≠ 0)
```

但直接编码需要 2^d 个电路实例，复杂度过高。因此建议对 d > 8 用采样法，d ≤ 8 用 SAT 或枚举。

### 3.4 基于电路的次数上界

对电路表示的 f，次数上界可以直接从电路中读出：

```
// 对每个门:
//  AND: deg(output) = deg(input1) + deg(input2)
//  XOR: deg(output) = max(deg(input1), deg(input2))
//  NOT: deg(output) = deg(input)
//  MUX: deg(output) = max(deg(sel)+deg(a), deg(sel)+deg(b), deg(a), deg(b))
```

**直接传播**得到每个输出分量的精确次数（假设 AND 是 F₂ 中的乘法，XOR 是加法）。这种传播在 O(gates) 时间内完成，完全免费。

### 3.5 完整的 ANF 重建（大规模）

已知依赖集 S 和次数上限 d 后，可以在 |S| 维子空间上重建 ANF：

```
1. 确定依赖集: dep_set = {j | f depends on x_j}
2. 确定次数: deg = degree_estimate(f, circuit_analysis)
3. 取 n' = min(|dep_set|, 一些可处理的值)
   若 n' ≤ 25: 在 dep_set 的变量上穷举重建 ANF
   若 n' > 25: 只保留 T=1 的单变量项（系数为 1 的变量）
              + SAT 检测高次项的存在性
```

## 4. 仿射变换的纤维丛理论

### 4.1 变换的几何结构

仿射变换 φ(x) = Mx ⊕ b 把 F₂ⁿ 划分为 ker(M) 的陪集（纤维）：

```
x ∼ x'  ⇔  φ(x) = φ(x')  ⇔  x ⊕ x' ∈ ker(M)
```

纤维数量 = 2^{rank(M)}，每条纤维大小 = 2^{n - rank(M)}。

### 4.2 G 存在的充要条件

**定理 1（常值纤维条件）：**

```
∃G: G∘φ = F  ⇔  ∀Δ ∈ ker(M), ∀x: F(x ⊕ Δ) = F(x)
```

**等价表述：**

```
∃G  ⇔  ker(M) ⊆ Aut(F),  其中
Aut(F) = {Δ ∈ F₂ⁿ | ∀x: F(x ⊕ Δ) = F(x)}
```

### 4.3 一致性验证算法

给定 M 和 truth table F_tt：

```
function is_consistent(M, F_tt, n):
  // 计算 ker(M) 的一组基
  basis = nullspace_basis(M, n)    // 高斯消元, O(n³)
  
  for each Δ in basis:
    // 检查 F 在 Δ 方向是否恒定
    // 即: 对所有 x, F_tt(x) == F_tt(x ⊕ Δ)
    if not constant_in_direction(F_tt, Δ, n):
      return False
  return True
```

`constant_in_direction` 的 TT 实现（n ≤ 20）：

```
function constant_in_direction(F_tt, Δ, n):
  n_words = ceil(2ⁿ / 64)
  for w = 0..n_words-1:
    if F_tt[w] != xor_shift(F_tt, Δ, w):
      return False      // 存在 x 使 f(x) ≠ f(x⊕Δ)
  return True
```

对大 n，用采样法代替全扫描。

### 4.4 不变子空间 Aut(F)

**定义：** Aut(F) 是 F₂ⁿ 的子空间，包含所有使 F 平移不变的 Δ。

**计算（小规模 n ≤ 20，Walsh 功率谱法）：**

```
对每个输出 f_i:
  1. Ŵ = Walsh-Hadamard(f_i)            O(n·2ⁿ)
  2. S[α] = Ŵ[α]² (为防溢出用浮点)      O(2ⁿ)
  3. r = Inverse_Walsh(S)               O(n·2ⁿ)
  4. Aut(f_i) = {Δ | r[Δ] = 1 - ε}      O(2ⁿ)

Aut(F) = ∩_i Aut(f_i)
```

**计算（大规模 n ≥ 26，SAT 法）：**

```
对每个输出 f_i:
  对一组候选 Δ（e_j 和随机组合）:
    SAT(∃x: f_i(x) ≠ f_i(x ⊕ Δ))
    若 UNSAT → Δ ∈ Aut(f_i)

  // 求得 Aut(f_i) 的一组基:
  // 从 {e_0,...,e_{n-1}} 开始, 检查每个是否在 Aut(f_i) 中
  // 再用基扩充法找完整子空间
```

### 4.5 性质：一致性的单调性

**定理 2：** 设 ker(M₁) ⊆ ker(M₂)。则：
- M₂ 一致 ⇒ M₁ 一致
- M₁ 不一致 ⇒ M₂ 不一致

**推论：**
- 加行（缩小核）保持一致性
- 删行（扩大核）可能破坏一致性
- 从 identity 出发（ker={0}，始终一致），加任何行都一致

### 4.6 维数下界

**定理 3：** m ≥ n - dim Aut(F)。下界可达当且仅当 ker(M) = Aut(F) 且 m = n - dim Aut(F)。

### 4.6.1 输出值数下界

m 还有一个来自信息论的独立下界：

**定理 3a（值数下界）：** 记 Image(F) = {F(x) | x ∈ F₂ⁿ} ⊆ F₂ᵏ，N = |Image(F)| 为 F 的不同输出值个数。则：

```
m ≥ ⌈log₂ N⌉
```

**证明：** G(z) 是 F₂ᵐ → F₂ᵏ 的函数，至多输出 2ᵐ 个不同值。而 F(x) = G(Mx⊕b) 输出 N 个不同值，故需 2ᵐ ≥ N。□

### 4.6.2 两个下界的关系

两个下界并不等价，但通过 Aut(F) 存在关联：

```
n - dim(Aut(F)) ≥ ⌈log₂ |Image(F)|⌉
```

**理由：** Aut(F) 的陪集将 F₂ⁿ 划分为 2^{n-dim(Aut(F))} 个等价类，F 在每个等价类上取常值。不同的输出值必然落于不同的等价类中，故等价类数 ≥ 输出值数。

**实用意义：** 对于搜索而言，两个下界需分别计算，取较大者。

**注意：** 计算 |Image(F)| 时只考虑非平凡输出分量。常数输出 (f_i(x) ≡ 0 或 ≡ 1)
在所有 x 处取值相同，不贡献区分信息。

```
m_min_abs = max(⌈log₂ N_nonconst⌉, n - dim(Aut(F))),  其中
N_nonconst = |{(f_{i1}(x),...,f_{ik'}(x)) | x∈F₂ⁿ}|,  仅非平凡分量参与
```

| 场景 | 非平凡输出数 | 值数下界 | Aut(F) 下界 | 实际 m_min |
|------|-------------|---------|------------|-----------|
| 常数输出 | 0 | 0 | n (单输出全空间) | 0 (无需搜索) |
| 单输出布尔函数 | 1 | 1 | n - dim(Aut(f)) | max(1, ...) |
| hd10 | 3 (29 个常数) | ⌈log₂ 8⌉ = 3 | ≈3×32 全依赖 → 约 32 | ≈32 |
| ctrl (7入26出) | 取决于电路 | ? | 取决于结构 | 待定 |

**注意：** 值数下界通常弱于 Aut(F) 下界（因为 Aut(F) 下界已蕴含它），但在以下情况独立有价值：
1. Aut(F) 不易计算时（n > 20），值数下界可通过采样快速估计
2. 输出值数极小（如大部分为常数）时，此下界提示搜索应从极小的 m 开始

## 4.7 3n 行池中 m 的下界与阈值

### 4.7.1 3n 行池结构

实际搜索中，M 的行从三类候选行构成的池中选取，总池大小 ≈ 3n：

```
行池 = {
  (1) identity 行:    e_i,          b=0    (i=0..n-1)      // n 行
  (2) complement 行:  e_i,          b=1    (i=0..n-1)      // n 行
  (3) XOR 对行:       e_i ⊕ e_j,    b=0    (前 k 个 Walsh 位) // ~n 行
}
```

从这 3n 行中选取 m 行构成 M ∈ F₂^{m×n} 和 b ∈ F₂^m。

### 4.7.2 m 的下界

**定理 4（下界定理）：** 对于给定的一致变换 (M,b)，记 rank(M) = r。若 M 的行从 3n 行池中选取，则：

```
m ≥ r ≥ n - dim Aut(F)
```

其中 r = rank(M) 是 M 的实际秩，m 是行数（可能大于 r 若存在冗余行）。

**实际确定 m_min 的方法：**
对给定输出 f_i：
1. 将行池按 Walsh 评分降序排列
2. 依次取前 1, 2, 3, ... 行构成 M
3. 调用 `evaluate_zspace` 检查一致性
4. 第一个得到 T < INT64_MAX 的 m 即为 m_min

**性质（单调性）：** 若行池固定且按评分排序，则一致性随 m 单调：
- 若 m = m₀ 得到一致变换，则对任意 m > m₀，取前 m 行仍一致（加行保持一致性）
- 若 m = m₀ 不一致，则对任意 m < m₀，取前 m 行仍不一致

**推论：** m_min 是判定一致性的最小所需行数，是搜索的起始点。

### 4.7.3 函数 T(m) 的行为

记 T(m) 为取前 m 行时得到的最佳 union_T（或 per-output T）。

**猜想（T(m) 的单调性）：** T(m) 随 m 增加单调不增。原因是加行增加 z 空间维度，额外的自由度允许 g 的 ANF 至少不劣于原有表达。

**实际观察到的三个阶段：**

```
T(m)
│
│  阶段 1         阶段 2          阶段 3
│  ┌─────┐       ┌──┐
│  │不一致│       │  │下降
│  │     │       │  │
│  │     │ ──── │  └────┐  ──── ──── ──── (平台期)
│  │     │/            │
│  └─────┴─────────────┴──────────────────▶ m
│  m_min              m_threshold
```

- **阶段 1 (m < m_min)：** 不一致，T = INT64_MAX
- **阶段 2 (m_min ≤ m < m_threshold)：** 一致，T 随 m 增加而显著下降
- **阶段 3 (m ≥ m_threshold)：** 平台期，T 不再显著下降

### 4.7.4 阈值判定准则

**阈值 m_threshold 的定义：** 增加 m 不再使 T(m) 有显著改善的最小 m。

**判定方法（经验准则）：**

```
给定连续 K 个 m 值: m₀, m₁, ..., m_{K-1}
若对相邻的 m, 均有 T(m_{t+1}) ≥ T(m_t) · (1-δ),
则判定已到达阈值。

其中:
  K = 3~5 (连续次数)
  δ = 0.01~0.05 (相对改善阈值)
```

即：若连续 K 次增加 m 后 T 的下降均不超过 δ×100%，则认为已达到阈值。

**替代终止条件：**
1. **冗余检测：** 后续行均通不过排他性检查（rank 不增加），此时添加行不改变 ker(M)
2. **Δ ∈ Aut(F)：** 后续行的 Δ 均在 Aut(F) 中，不提供新的区分方向
3. **T=0 终止：** 若 T(m) = 0，已达绝对最优，提前终止

### 4.7.5 m 的搜索策略对比

**自底向上策略（推荐）：**

```
1. 行池排序 (Walsh 评分降序)
2. m = 1, 2, 3, ... 递增:
   a. 取前 m 行作为候选 M
   b. 计算 T(m)
   c. 若 T(m) = 0: 终止 (最优)
   d. 若 m ≥ m_min 且 T(m) 连续 K 次不改善: 终止 (达到阈值)
```

**优势：**
- 从最小 m 开始，每次只增加一行，计算量最小
- 单调一致性保证：不会走了弯路
- 阈值判定节省大 m 时的计算

**与现有策略的对比：**

| 策略 | 起始 m | 方向 | 终止条件 | 适用场景 |
|------|--------|------|----------|---------|
| 自底向上 | m=1 | 增加 | 阈值判定 | 大 n 未知结构 |
| 渐进构造 | m=n | 增加 | 池耗尽 | 小 n 系统搜索 |
| 随机候选 | 随机 m | — | 候选数用完 | 快速探索 |

### 4.7.6 两个优化器的理论适配

目前代码中有两个优化器，理论框架对它们的适配方式不同：

**优化器 1: truth-table (search.cpp, n ≤ 20)**

| 理论要素 | 实现方式 |
|---------|---------|
| Aut(F) 计算 | Walsh 功率谱法（精确） |
| m_min | m_min = n - dim(Aut(F))，精确已知 |
| 行池 | 3n 行 (build_row_pool) + Walsh 单行 + 随机 |
| 搜索方向 | 从 m=n 向下（渐进构造删除行）或从 m=m_min 向上 |
| 评估方式 | evaluate_Mb（TT 交换 O(m·2^m)） |

由于 Aut(F) 精确已知，m_min 可直接算得。搜索可以从 m_min 开始向上试探，每步增加一行并计算 T(m)。因 T(m) 单调不增（加行可保留旧 g），阈值检测可靠。

**优化器 2: z-space (main_large.cpp, n > 20)**

| 理论要素 | 实现方式 |
|---------|---------|
| Aut(F) 计算 | 不可行（需 2^n 空间），返回空 |
| m_min | 未知，需通过行池试探 |
| 行池 | 3n 行 + 随机行 + 密集随机行 |
| 搜索方向 | 当前：各阶段不同（随机 m） |
| 评估方式 | evaluate_zspace（电路求值 O(m·2^m · gates)） |

由于 Aut(F) 未知，m_min 无法先验确定。需通过构建行池并按 Walsh 评分排序后，自底向上试探第一个得到一致变换的 m。

**定理 5（单调性保持，z-space）：** 在 z-space 评估中，若 M_{m+1} = [M_m; w]（第 m+1 行追加），则可在 z_{m+1} 上定义 g_{m+1} 使其不依赖 z_{m+1}，从而 T(m+1) ≤ T(m)。因此 z-space 中 T(m) 也单调不增。

### 4.7.7 Walsh 评分的理论依据

**定义：** 对候选行 w ∈ F₂ⁿ，其 Walsh 评分定义为：

```
Score(w) = Σ_{i=1..k} |Ŵ_i(w)| / 2ⁿ
```

其中 Ŵ_i(w) = Σ_x (-1)^{f_i(x) ⊕ ⟨w,x⟩} 是输出 f_i 在 w 方向的 Walsh 系数。

**理由：** |Ŵ(w)| 大意味着 f(x) 与线性函数 ⟨w,x⟩ 有强相关性。当 w 作为 M 的一行时，z = ⟨w,x⟩ ⊕ b（b 为 0 或 1），强相关性意味着 z 携带了大量关于 f 的信息，使 g(z) 更可能稀疏。

**局限性：** Walsh 评分是单行独立的度量，不反映多行组合效应。两行各自有高 Walsh 评分，但组合在一起可能冗余。弥补方式：
1. 排他性检查（去除线性相关行）
2. Aut(F) 基检查（去除冗余方向）
3. 逐 m 评估 T(m) 可检测组合效果

## 5. 搜索 M 使 G 存在的基本策略

### 5.1 完全搜索不可行

对 n=32，可能的 M 数量约为 Σ_{m=1}^{32} C(2³², m) ≈ 2^{32×32} = 2^{1024}。穷举不可行。

### 5.2 降维原则

以下条件确保 ker(M) ⊆ Aut(F)，即 G 存在：

```
(1) ker(M) = {0}          ← M 列满秩, m=n 且 rank=n
(2) M 的行向量 ⊆ V        ← V 是 Aut(F) 补空间的行表示
(3) ker(M) = Aut(F)       ← 最优压缩, 达到下界
```

**实际操作：** 取 Aut(F) 的补空间 V（dim n' = n-d），在 V 上搜索 M 的行。

### 5.3 渐进式构造

```
M_curr = []    // 空
basis = {e_0,...,e_{n-1}}  // ker(M_curr) 的基

while |M_curr| < max_m:
  对每个候选行 w:
    // 步骤 1: 排他性检查
    if rank(M_curr ∪ [w]) == rank(M_curr):
      continue          // w 的行向量在 M_curr 的行空间中，冗余
    
    // 步骤 2: 找 Δ 使 M_curr·Δ=0, w·Δ=1
    增广矩阵 [M_curr; w]·Δ = [0; 1]
    高斯消元求特解 Δ
    if 无解: continue   // w 不提供新方向
    
    // 步骤 3: 检查 Δ 是否在 Aut(F) 中
    if Δ ∈ Aut(F): continue  // w 划分了无效方向
    
    // 步骤 4: 接受 w
    M_curr ∪= [w]
    更新 ker(M_curr) 的基
```

### 5.4 候选行评分

对通过排他性检验的候选行 w，需要评分选择加入：

```
// Walsh 评分（已有）
score = Σ_{oi} |W_oi(w)|  // 各输出在 w 方向的 Walsh 幅值之和

// 差分评分（新）
score = Σ_{oi} Prob_x[f_oi(x) ≠ f_oi(x ⊕ Δ)]
      其中 Δ 是步骤 2 解出的特解（M_curr·Δ=0, w·Δ=1）

// 稀疏性评分（直接估计 T(g)）
score = -E[T(g_oi)]       // 估计新 ANF 的单项式数的变化
```

实际中 Walsh 评分最实用——可直接从 Walsh 谱得到，无需额外电路求值。

## 6. 优化 g 的 ANF 稀疏性

### 6.1 T(g) 的决定因素

对单输出 f 和变换 z = Mx⊕b (m=n)，g(z) = f(M^{-1}(z⊕b))。T(g) = T(f)（Möbius 项数在可逆变换下不变——实际上这不对。T(f) 是 ANF 项数，而 ANF 项与变量命名有关，可逆仿射变换会改变哪些变量名但项数不变）。

等等，仔细想：如果 M 可逆，g(z) = f(M^{-1}(z⊕b))，g 的 ANF 是 f 的 ANF 做变量代换 x = M^{-1}z⊕(M^{-1}b)。变量代换不改变项数（每个单项式映射到另一个单项式，次数不变，一一对应），所以 T(g) = T(f)。

**所以可逆 M 无法压缩 ANF。真正的压缩只来自 m < n（投影退化）。**

### 6.2 互补对滤波

m > n 时，如果 z_i 和 z_j 是互补对（z_i = x_k, z_j = x_k ⊕ 1），则 z_i·z_j = 0（因为 x_k(x_k⊕1) = 0）。在莫比乌斯变换后，所有包含互补对的项都可以滤除。

**实现：**

```
1. 对 g 做 Möbius → ANF 系数 a
2. 对每个互补对 (i,j):
   删除所有包含 i 和 j 的项
3. 对剩余系数做逆 Möbius → 新的 g_tt
```

这等价于在 ANF 系数层面施加约束 z_i·z_j = 0。滤除后 T(g) 可能大幅下降。

### 6.3 依赖集缩减

对策略 2（每输出独立 M），筛选仅保留依赖集内的输入变量：

```
M_i 的行只在 D_i = {j | e_j ∉ Aut(f_i)} 内有 1。
```

此时各行支撑集在 n' = |D_i| 维上。T(g) 的上界显著降低。

### 6.4 联合 vs 独立策略的权衡

**策略 1（联合 M）：** 所有输出共享 M
- Aut(F) = ∩ Aut(f_i)，更小
- m 下界更大
- 各输出互相影响，T(g) 可能增大

**策略 2（独立 M_i）：** 各输出有各自 M_i
- Aut(f_i) 各自，更大
- 各 m_i 下界更小
- 合并后 z 是各 m_i 之和，可能很大

**混合：** 分组共享（共享部分 M 行）

### 6.5 广义仿射映射与稀疏 g 枚举法

#### 6.5.1 广义仿射映射框架

前面（第 4.2 节）讨论的 G 存在条件要求 f(x) ≡ G(Mx⊕b)。更一般地，允许输出侧附加一个 x 的仿射函数：

```
f(x) = g(Mx⊕b) ⊕ ⟨c, x⟩ ⊕ d
```

其中：
- M ∈ F₂^{m×n}：输入侧仿射映射（当前框架已有）
- b ∈ F₂^m：输入侧偏置
- c ∈ F₂^n：输出侧线性项系数
- d ∈ F₂：输出侧常数项

⟨c,x⟩ = c₁x₁ ⊕ c₂x₂ ⊕ ... ⊕ cₙxₙ 是 x 的线性函数。

**与现有框架的关系：** 当 c=0, d=0 时退化为 f(x)=g(Mx⊕b)，是当前实现的特例。

#### 6.5.2 广义一致性的理论推导

给定 f: F₂ⁿ → F₂ 和参数 m，目标是寻找 (M,b,c,d,g) 满足：

```
f(x) = g(Mx⊕b) ⊕ ⟨c,x⟩ ⊕ d          (1)
```

对任意 Δ ∈ ker(M)，代入 x 和 x⊕Δ：

```
f(x)     = g(Mx⊕b)     ⊕ ⟨c,x⟩     ⊕ d
f(x⊕Δ)  = g(Mx⊕b)     ⊕ ⟨c,x⊕Δ⟩   ⊕ d     (因为 MΔ = 0)
         = g(Mx⊕b)     ⊕ ⟨c,x⟩ ⊕ ⟨c,Δ⟩ ⊕ d
```

两式 XOR：

```
f(x) ⊕ f(x⊕Δ) = ⟨c,Δ⟩                 (2)
```

**关键结论：** f(x)⊕f(x⊕Δ) 必须是与 x 无关的常数（对每个 Δ∈ker(M) 各自独立）。

这就是**广义一致性条件**。对比传统条件：

| 框架 | 条件 | 物理意义 |
|------|------|---------|
| 传统 f = g∘φ | f(x) ≡ f(x⊕Δ) | f 在 Δ 方向平移不变 |
| 广义 f = g∘φ⊕⟨c,x⟩⊕d | f(x)⊕f(x⊕Δ) ≡ const | f 在 Δ 方向一阶差分为常数 |

广义条件更宽松。传统要求 f 在 ker(M) 上完全常数，广义只要求 f 在 ker(M) 上的**一阶差分是常数**。

#### 6.5.3 与现有算法的关系（重要区别）

**关键差异：** 现有的"仿射等价"算法均研究 A ∈ GL(n,F₂) 即可逆方阵的情况，而我们处理的是 M ∈ F₂^{m×n} 的一般矩阵（m < n，非方阵、不必满秩）。两者是不同的数学问题。

```
传统仿射等价:  f(x) = g(Ax+b),    A∈GL(n,F₂),  n→n 可逆
我们的仿射映射:  f(x) = g(Mx+b)⊕⟨c,x⟩⊕d,   M∈F₂^{m×n}, n→m 降维
```

因此以下算法**不能直接使用**，仅可作为理论参考：

| 算法 | 适用条件 | 为何不直接适用 |
|------|---------|--------------|
| D-可约函数 (Ciriani) | A 将 f 投影到仿射子空间 | 概念相通但需输出侧 ⟨c,x⟩ 扩展 |
| On-set 矩阵法 (电子科技大学) | 可逆 A 下秩不变 | M 非方阵时 On-set 矩阵的秩变化无简单规律 |
| EA-等价恢复 (Canteaut 2022) | A 可逆, B 可逆, deg≤2 | 仅限二次 + 可逆 |
| Guess-and-Determine (Biryukov) | A 可逆, 双射 | 不适用（M 不可逆）|

**与 D-可约函数的联系（最接近）：** D-可约函数 f 的所有 1-点包含在 dim < n 的仿射空间 A 中，因而 f 可投影到 A 上。在我们的框架中，若 ⟨c,x⟩ 将 f 校正为 f'(x) = f(x)⊕⟨c,x⟩ 后，f' 在 ker(M) 的每个陪集上为常数，则 f' 在 ker(M) 方向 D-可约。这比标准 D-可约多了一个"输出侧仿射校正"的自由度。

**结论：** 由于 M 是降维映射、存在非平凡核空间，我们的问题不能直接归约为传统仿射等价。第 6.5.2 节推导的广义一致性条件和第 6.5.4 节的枚举算法是针对一般矩阵的正确方法。

#### 6.5.4 完整判定与计算算法

```
输入: f 的 truth table（n ≤ 16），参数 max_m, K（池大小）
输出: 最优 (M,b,c,d,g,T(g))

对 m = 1..max_m:
  行池 ← 取 3n 池中前 K 行（Walsh 评分前 top）
  对所有 C(K,m) 种 (M,b) 组合:
    
    阶段 A — 广义一致性检查
    ker_basis = nullspace_basis(M, n)   // 高斯消元
    for 每个 Δ ∈ ker_basis:
      ref = f(0) ⊕ f(Δ)                // 参考值
      for all x ∈ F₂ⁿ:
        if f(x) ⊕ f(x⊕Δ) ≠ ref:
          goto NEXT_MB                  // 不满足
    
    阶段 B — 解 c
    // 对每个基 Δⱼ: ⟨c, Δⱼ⟩ = f(0)⊕f(Δⱼ)
    c = solve_linear(B, const_vector)
    // c 在 ker(M)⊥ 的分量设为 0
    
    阶段 C — 构造 g
    M⁺ = right_inverse(M, m, n)
    g_tt  ← 大小为 2ᵐ 的数组
    for each z ∈ F₂ᵐ:
      x = M⁺·(z ⊕ b)
      g_tt[z] = f[x] ⊕ ⟨c, x⟩
    
    阶段 D — 计算 T(g)
    moebius(g_tt, m)
    T = popcount(g_tt)
    T = min(T, 2ᵐ - T)          // 含 d 选择
    if T < best_T:
      best = (M,b,c,d,g_tt,T)
    
    NEXT_MB: continue

输出 best
```

#### 6.5.5 与当前随机 M,b 搜索的关系

| 方法 | 适用 m | 最优性保证 | 成本 |
|------|--------|-----------|------|
| 本算法（3n 池穷举） | m ≤ 5 | **保证**（遍历池内组合） | C(K,m) × O(dim·2ⁿ) |
| 当前随机 M,b | 任意 m | 无 | n_random × O(2ᵐ) |

**互补使用：** m ≤ 5 用本算法保证最优，m > 5 用随机搜索覆盖更广。

### 8.1 hd08 (n=8, k=1)

- Aut(F): Walsh 法计算，假设 dim=0（非线性函数无平移不变性）
- m_min = 8 - 0 = 8，但已知存在压缩到 m=6
- **矛盾解释：** m < n 时 rank(M) ≤ m。若 rank(M)=6，ker(M) 维数=2。一致性需 ker(M) ⊆ Aut(F)。但 Aut(F) 维数=0 意味着 ker(M) 必须为 {0}，即 rank(M)=8。
  
  **结论：** hd08 的 m=6 变换必须是**不一致的**（G 不完全等于 F 的某个分量），或在某个我们没有考虑的近似意义下成立。

  等等，但这不可能。我们的代码中 `evaluate_Mb_single_output` 会检查一致性，不一致时返回 INT64_MAX。hd08 结果确实是通过验证的。

  再检查：hd08 的 m=6 解是 `hill_climb` 从 identity 出发找到的。但 hill_climb 用的是 `evaluate_Mb` 中的 `hill_climb`，它作用于 m=n 的 M（修改 M 的行但保留 m=n）。所以 hill climb 后的 M 仍是 m=n，只是某些行可能变成全 0 → rank < 8。

  m=n=8，但 rank(M)=6？不对，m 仍为 8。如果 rank(M)=6，则 ker(M) 维数=2。一致性要求这 2 维在 Aut(F) 中。但 Aut(F) 维数=0...

  实际上，对 hd08 函数（8 输入单输出），Aut(F) 可能不是 {0}。让我重新思考：如果 Aut(F)={0}，那么任何非零 Δ 都会改变至少一个 x 处的 f 值。对于完全随机函数确实如此。但 hd08 是一个有结构的电路函数（39 门），可能具有非平凡的不变性。

  或者更可能的解释：hill climb 从 identity (m=n=8) 出发，修改了 M 的行但维持 m=8。得到的变换具有 ker(M)={Δ|MΔ=0}。如果 ker(M) 的维数 > 0，则需 ker(M) ⊆ Aut(F)。也许 hd08 本身就有 Aut(F) ≠ {0}。

  关键点：**我们之前从未计算过实际的 Aut(F) 值**。这只是理论分析。

- 对常数输出：Aut(f_i) = F₂³²（全空间），dim=32
- 对非平凡输出：依赖全部 32 个输入（通过 AND/XOR/NOT 级联），测试发现 m<n 时右逆固定未映射位导致不一致
- 联合 Aut(F) = ∩_{非平凡} Aut(f_i) ∩ F₂³² = ∩_{非平凡} Aut(f_i)
- 由于非平凡输出覆盖所有输入变量，dim(Aut(F)) 很小，联合 m_min ≈ n
- Python opt2 得到 T=16 使用排列 M（每输出单独排列，不是联合策略）

## 9. 理论-算法对照表

| 理论问题 | 小规模算法 (n≤20) | 大规模算法 (n≥26) |
|---------|-----------------|-----------------|
| T(f) 计算 | Möbius O(n·2ⁿ) | 依赖集 + 次数检测 |
| Aut(F) | Walsh 功率谱 | SAT 逐基检测 |
| 依赖集 D(F) | TT 平移 e_j 扫描 | 采样 / SAT |
| deg(F) | 直接读 ANF | 电路传播 / 采样 |
| ker(M) ⊆ Aut(F)? | TT 扫描所有 Δ | 采样 / SAT |
| 排他性 Δ | 高斯消元 (同) | 高斯消元 (同) |
| G 的 ANF | Möbius O(m·2^m) | z-space 求值 |
| 验证 | 穷举 2ⁿ | 随机点 N=5000 |

## 10. 开放问题

1. **hd08 示例的本理解释：** 我们的 hill climb 找到了 m=8 的变换使 T(f) 从 128 降到 6。这个 M 的 ker(M) 和 Aut(F) 的关系如何？

2. **非满秩 M 何时保持一致性：** 当 rank(M) < m 时，额外的 m-rank(M) 行是纯冗余的（可被其他行线性表示）。在这种情况下，理论是否允许更大的压缩？

3. **互补对滤波的理论** 等价于在 Boole 环中添加约束 z_i·z_{n+i}=0。这个操作可能产生原本不可达的稀疏解？还是等价于某个仿射变换空间中的特定点？

4. **G 的唯一性：** 给定一致的 (M,b)，G 是否唯一？还是同一 (M,b) 对应多个不同 G（在不可达的 z 点上取值不同）？

### 11.3 组 B 实例分析 (n > 20)

组 B 实例共有的挑战：
1. **Aut(F) 不可计算**：Walsh 功率谱法需要 O(n·2ⁿ) 内存和时间，n=32 时 2³² = 4G 点不可行
2. **m_min 不可先验确定**：因 Aut(F) 未知，无法直接计算 m_min = n - dim(Aut(F))
3. **评估代价高**：2^m 次电路求值，m=32 时不可行

**组 B 的通用策略：** 每输出独立搜索（策略 2），自底向上从 m=1 开始试探。

#### 11.3.1 hd10 (n=32, k=32, gates=121)

**电路特征：** 32 输入 32 输出，121 门。29 个常数 0 输出 + 3 个非平凡输出 (m_29, m_30, m_31)。

| 项目 | 值 |
|------|-----|
| 已知最优 T | 16 (Python opt2，排列 M) |
| 非平凡输出 | m_29, m_30, m_31（依赖全部 32 输入） |
| Image(F) 大小（仅非平凡） | ≤ 8 |
| 值数下界 | ⌈log₂ 8⌉ = 3 |
| Aut(F) 下界 | ≈32（非平凡输出全依赖） |
| 实际 m_min | m_29/m_30/m_31 需 m=32（identity），但评估 2³² 点不可行 |
| 结论 | **大优化器无法压缩**非平凡输出。需用 optimize_anf（truth-table）或接受 identity |
| 搜索策略 | 对 29 个常数输出：m=1 即 T=0（Phase 1 发现）|
| 验证 | 随机 5000 点 |

**关键决策：** hd10 的非平凡输出在大优化器中无法压缩。改造策略：
1. **选项 A**：对非平凡输出跳过，只报告常数输出的结果
2. **选项 B**：使用 optimize_anf（n=32 truth-table，但只需对非平凡输出做 TT）
3. **选项 C**：如果非平凡输出失败，报错退出（当前实现）

**推荐：选项 A**——报告常数输出结果，非平凡输出标记为 "无法优化"。

#### 11.3.2 hd01 (n=32, k=32, gates=121)

**电路特征：** 32 输入 32 输出，121 门。每个输出 T(f) = 1（常数或单变量 x_j）。

| 项目 | 值 |
|------|-----|
| 已知最优 T | 32 (Python opt2，排列 M) |
| 原始 T(f) | 32（每输出 T=1）|
| 每个输出依赖 | 至多 1 个输入变量 |
| dim(Aut(f_i)) | 31（排除 e_j）|
| m_min_i | 1（只需一个 identity 行）|
| 搜索策略 | Phase 1 identity 行直接找到 m=1, T=0 或 T=1 |
| 预期 T | 32（m=1 per-output，union T = 32）|
| 预期运行时间 | < 1 min |
| 验证 | 随机 5000 点 |

**预期结论：** 大优化器 Phase 1 直接找到每输出的 m=1 变换（因为各输出只依赖一个变量，identity 行直接匹配）。结果应与 Python opt2 相同（T=32）。

#### 11.3.3 hd02 (n=32, k=32, gates=231)

**电路特征：** 32 输入 32 输出，231 门。每输出 T(f) = 1~4。

| 项目 | 值 |
|------|-----|
| 已知最优 T | 64 (Python opt2，排列 M) |
| 原始 T(f) | 约 64（每输出 T=1~4）|
| 每个输出依赖 | 少量输入变量 |
| dim(Aut(f_i)) | 接近 32 |
| m_min_i | 很小（1~4） |
| 搜索策略 | Phase 1 identity 行 + Phase 2 XOR 对 |
| 预期运行时间 | 几分钟 |
| 验证 | 随机 5000 点 |

**预期结论：** 类似 hd01，但各输出可能依赖多个变量，需要 XOR 对（Phase 2b）找到最佳单行组合。预期结果接近 Python opt2（T=64）。

### 11.5 失败模式与处理

| 失败模式 | 检测方式 | 处理方式 |
|---------|---------|---------|
| 一致性检查失败 | evaluate_zspace 返回 INT64_MAX | 跳过该候选，继续搜索 |
| 验证失败 | verify_candidate 返回 false | 标记为无效，从结果中移除 |
| 所有候选均不一致 | best.total_T = INT64_MAX | 使用 identity 回退（n≤24）或报错退出 |
| 运行时间过长 | 自定义 timeout | 截断搜索，使用当前最优 |
| 内存不足 | 系统 OOM | 减小 max_m 或 n32_random |
